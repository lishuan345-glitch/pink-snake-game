# 贪吃蛇游戏开发规范

## 1. 项目概述
本项目是一个HTML网页端的贪吃蛇小游戏，采用粉红色可爱风格设计，配有轻快的电子音乐背景。

## 2. 技术栈
- **HTML5**: 页面结构
- **CSS3**: 样式和动画
- **JavaScript (ES6+)**: 游戏逻辑
- **Canvas API**: 游戏渲染
- **Web Audio API**: 音频处理

## 3. 代码结构规范

### 3.1 文件组织
```
贪吃蛇/
├── index.html          # 主HTML文件（包含所有代码）
├── claude.md           # 开发规范文档
└── README.md           # 项目说明文档（可选）
```

### 3.2 代码组织顺序
在单文件开发中，代码按以下顺序组织：
1. HTML文档结构
2. CSS样式（放在 `<style>` 标签中）
3. JavaScript代码（放在 `<script>` 标签中）

## 4. 命名规范

### 4.1 变量命名
- 使用小驼峰命名法：`camelCase`
- 常量使用全大写：`SNAKE_SPEED`
- 布尔变量以 `is`、`has`、`can` 开头：`isGameOver`

### 4.2 函数命名
- 使用小驼峰命名法：`camelCase`
- 动词开头：`drawSnake()`, `updateGame()`, `handleKeyPress()`
- 事件处理函数以 `on` 或 `handle` 开头：`onKeyDown()`, `handleCollision()`

### 4.3 CSS类命名
- 使用短横线命名法：`kebab-case`
- 语义化命名：`.game-container`, `.score-board`, `.start-button`

## 5. 代码风格

### 5.1 JavaScript风格
- 使用 `const` 和 `let`，避免使用 `var`
- 函数优先使用箭头函数，除非需要 `this` 绑定
- 使用模板字符串代替字符串拼接
- 每个函数功能单一，不超过50行
- 适当使用空行分隔逻辑块

### 5.2 注释规范
- 文件头部添加整体说明注释
- 每个主要函数添加功能说明注释
- 复杂逻辑添加行内注释
- 注释使用中文，清晰易懂

**注释示例：**
```javascript
/**
 * 绘制蛇的身体
 * @param {Array} snake - 蛇的坐标数组
 */
function drawSnake(snake) {
  // 遍历蛇的每个节点
  snake.forEach((segment, index) => {
    // 蛇头和身体使用不同颜色
    ctx.fillStyle = index === 0 ? '#FF69B4' : '#FFB6C1';
    ctx.fillRect(segment.x, segment.y, CELL_SIZE, CELL_SIZE);
  });
}
```

## 6. 游戏设计规范

### 6.1 游戏元素
- **蛇头**: 深粉色 (#FF69B4)
- **蛇身**: 浅粉色 (#FFB6C1)
- **食物**: 亮粉色带光晕效果 (#FF1493)
- **背景**: 柔和的粉白渐变
- **网格线**: 淡粉色虚线（可选）

### 6.2 游戏参数
```javascript
const CANVAS_WIDTH = 600;      // 画布宽度
const CANVAS_HEIGHT = 600;     // 画布高度
const CELL_SIZE = 20;          // 单元格大小
const INITIAL_SPEED = 150;     // 初始速度（毫秒）
const INITIAL_LENGTH = 3;      // 蛇的初始长度
```

### 6.3 游戏功能
必须实现的核心功能：
1. 蛇的移动（上下左右）
2. 食物随机生成
3. 碰撞检测（墙壁、自身）
4. 分数统计
5. 游戏暂停/继续
6. 游戏重新开始
7. 背景音乐控制（播放/暂停）

## 7. UI/UX设计规范

### 7.1 配色方案
- 主色调: 粉红色系
  - 深粉: #FF69B4 (Hot Pink)
  - 中粉: #FFB6C1 (Light Pink)
  - 亮粉: #FF1493 (Deep Pink)
  - 淡粉: #FFF0F5 (Lavender Blush)
- 文字颜色: #FF1493 或 #FF69B4
- 按钮渐变: 粉色到紫色

### 7.2 可爱元素
- 圆角设计（border-radius）
- 柔和阴影（box-shadow）
- 渐变背景
- 可爱字体（如圆体）
- 动画效果（hover、active状态）

### 7.3 交互反馈
- 按钮悬停效果
- 游戏结束时的提示动画
- 得分时的视觉反馈
- 流畅的过渡效果

## 8. 音频规范

### 8.1 背景音乐
- 格式: MP3/OGG
- 时长: 循环播放
- 音量: 默认50%，可调节
- 风格: 可爱的电子音乐

### 8.2 音频控制
- 提供播放/暂停按钮
- 音量控制滑块（可选）
- 页面加载后自动播放（需用户交互）

## 9. 性能优化

### 9.1 Canvas优化
- 使用 `requestAnimationFrame` 进行动画
- 只重绘变化的区域（如可能）
- 避免频繁的DOM操作

### 9.2 代码优化
- 避免全局变量污染
- 使用事件委托
- 及时清理定时器和事件监听器
